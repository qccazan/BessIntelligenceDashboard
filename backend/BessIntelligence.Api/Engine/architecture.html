☀️ SolarProductionForecaster — FastTree Regression

Predicts solar panel output (kW) from weather conditions and panel specs. Trained on ~70,000 historical 15-minute production records (D-09) joined with weather (D-06) and installation specs (D-08). Uses 150 FastTree decision trees with 30 leaves each. Outputs a point estimate with a confidence interval (±15% clear sky, ±30% cloudy).

📥 Input Features

SolarIrradianceWm2420W/m² from weather forecast
CloudCoverPct25% cloud cover
AmbientTempC18.5°C ambient temperature
WindSpeedMs7.2m/s — helps cool panels
HourOfDay120–23 — sun angle proxy
Month61–12 — seasonal pattern
CapacityKwp350Panel peak capacity (kWp)
TiltDeg25Panel tilt angle
AzimuthDeg180Panel facing direction
Latitude52.1Site latitude (Netherlands)

📤 Example: Clear Summer Noon

GHI 420 W/m², cloud 25%, 350 kWp panel
→ ProductionKw = 280 kW
→ ConfidenceLow = 238 kW (−15%)
→ ConfidenceHigh = 322 kW (+15%)

📤 Example: Cloudy Winter Morning

GHI 80 W/m², cloud 75%, 350 kWp panel
→ ProductionKw = 42 kW
→ ConfidenceLow = 29 kW (−30%)
→ ConfidenceHigh = 55 kW (+30%)
⚠️ Cloud > 60% widens confidence interval to ±30%

📤 Example: Night

GHI 0 W/m², HourOfDay 2
→ ProductionKw = 0 kW (clamped to 0)

🔍 AnomalyDetector — SSA Spike Detection

Detects abnormal State-of-Charge (SoC) patterns per battery using Singular Spectrum Analysis. Trained per battery on 7-day SoC history (D-03b) — one model per battery stored in memory. Uses a 3-day training window with 96-point daily seasonality (15-min intervals × 24h). Output is a score from 0.0 (normal) to 1.0 (every reading anomalous) — the fraction of the last 96 points flagged as spikes.

📥 Input

batteryId: 1 (identifies which model to use)
socSeries: Array of SocTimePoint { SocPct }
672 points = 7 days × 96 intervals/day
SSA params: confidence 95%, pValueHistory 48,
trainingWindow 288 (3 days), seasonality 96 (1 day)

✅ Healthy — score 0.0

Smooth daily cycle: charge → 85%, discharge → 20%
Spikes detected in last 96 points: 0
→ AnomalyScore = 0 / 96 = 0.00
Battery fully eligible for dispatch

⚠️ Minor — score 0.03

Brief 5% SoC drop mid-charge (grid blip)
Spikes detected in last 96 points: 3
→ AnomalyScore = 3 / 96 = 0.031
Low risk — transient glitch, still dispatched

🔶 Moderate — score 0.10

SoC jumps ±10–15% erratically during charge
Spikes detected in last 96 points: 10
→ AnomalyScore = 10 / 96 = 0.104
Possible BMS issue — reduced dispatch participation

🔴 Severe — score 0.25

SoC drops 70% → 30% in 1h, then oscillates
Spikes detected in last 96 points: 24
→ AnomalyScore = 24 / 96 = 0.250
Major fault risk — battery held from dispatch, maintenance flagged

📉 DegradationPredictor — FastTree Regression

Predicts the cost per charge/discharge cycle (€) from battery operating conditions. Replaces the hardcoded formula (DoD/100)^1.3 × factors with a learned non-linear model. Trained on historical SoH trajectories from D-03b using 80 FastTree decision trees with 15 leaves. Falls back to the deterministic formula when insufficient training data (<50 samples).

📥 Input Features

DodPct35Depth of discharge %
CRate0.5Charge/discharge rate (C)
AmbientTempC22°C ambient temperature
SohPct97State of health %
TimeAbove80Pct0.12Fraction of time SoC > 80%
CapacityKwh1000Battery nameplate capacity

DoD 35%, C-rate 0.5, SoH 97%, temp 22°C
→ ML: €8.20/cycle
→ Fallback: €8.66/cycle
Shallow cycle, moderate rate — minimal wear

🟡 Moderate-Wear Cycle

Deeper cycle + higher C-rate increases cost

🔴 High-Wear Cycle

DoD 90%, C-rate 1.0, SoH 83%, temp 38°C
→ ML: €42.60/cycle
→ Fallback: €45.30/cycle
Near-full cycle + degraded battery + heat penalty

📐 Deterministic Fallback Formula

BaseCost = 0.030 EUR/kWh // (120 EUR/kWh replacement ÷ 4000 cycles)
DodFactor = (DoD / 100) ^ 1.3
CRateFactor = 1.0 + max(0, CRate − 0.5) × 0.30
TempFactor = <5°C → 1.25 | >35°C → 1.20 | else → 1.00
CalendarFactor = timeAbove80% <10% → 1.00 | ≤25% → 1.08 | else → 1.15
SoH_Mult = 100 / SoH

Cost = Base × Capacity × DoD × CRate × Temp × Calendar × SoH
