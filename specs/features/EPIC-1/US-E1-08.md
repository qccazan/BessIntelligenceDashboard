# US-E1-08 · Engine Configuration Constants (D-07)

## User Story

**As a** developer building the PoC data layer, **I want to** provide hardcoded accuracy constants that stand in for external metrics required by the F-06 confidence scoring step, **so that** the confidence formula is resolvable and produces the target 85% confidence for the PoC dataset.

## Description

Hardcoded PoC constants stored in `engine_config.json`. In production these would be computed from trailing error logs against KNMI and EPEX actuals. For the PoC they are static values tuned to produce confidence_pct = 85.

| Constant | Step | PoC value | Gives score |
|----------|------|-----------|-------------|
| price_MAE_pct | 6a — Price forecast accuracy | 4.0 | 80 |
| wind_RMSE_ms | 6a — Wind forecast confidence | 1.5 | 88 |
| cloud_MAE_pct | 6a — Solar forecast confidence | 9.0 | 82 |
| sigma_soh_pct | 6a — Fleet SoH uniformity | 0.3 | 97 |
| avg_30d_spread_multiplier | Step 7 — Explanation template | 6.1 | n/a |

Verification: round(0.40×80 + 0.25×88 + 0.20×82 + 0.15×97) = round(32+22+16.4+14.6) = 85.

## Acceptance Criteria

- **AC-1**: The config file contains all five constants: price_MAE_pct, wind_RMSE_ms, cloud_MAE_pct, sigma_soh_pct, avg_30d_spread_multiplier.
- **AC-2**: price_MAE_pct = 4.0 (produces price_forecast_score = 80).
- **AC-3**: wind_RMSE_ms = 1.5 (produces wind_forecast_score = 88).
- **AC-4**: cloud_MAE_pct = 9.0 (produces solar_forecast_score = 82).
- **AC-5**: sigma_soh_pct = 0.3 (produces fleet_soh_score = 97).
- **AC-6**: avg_30d_spread_multiplier = 6.1.
- **AC-7**: Using these constants in the F-06 confidence formula produces confidence_pct = 85 for a Wind+Solar day.
