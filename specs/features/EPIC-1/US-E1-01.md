# US-E1-01 · Past-Year Historical Data Generation (REQ-HIST-01)

## User Story

**As a** developer seeding the PoC database, **I want to** generate synthetic data covering a continuous rolling window of the past 12 months for all datasets (D-01 through D-07), **so that** the Intelligence Layer has sufficient historical depth for trend analysis, performance benchmarking, and advisory report generation.

## Description

For each dataset defined in the mock data specification, the system must generate synthetic data covering a continuous rolling window of the past 12 months (365 days), ending at the current date. All historical records must be internally consistent with the cross-dataset consistency rules. Time series data must use UTC-offset timestamps (Europe/Amsterdam, CET/CEST) and must not contain gaps, duplicate timestamps, or out-of-order entries.

Per-dataset volume requirements:
- D-01 Battery Master Data: Static per asset; commissioning dates within the past year where applicable.
- D-02 Real-Time Telemetry: One snapshot per hour 15 min for the past year as archive; live snapshot is most recent.
- D-03a Battery History 24h: 96 intervals per asset — consumed by F-05 Replay Card.
- D-03b Battery History 7d: 672 intervals per asset (96 × 7 days) — consumed by F-06 engine.
- D-04 Day-Ahead Market Prices: 24 prices/day × 365 days = 8,760 hourly records.
- D-05 AI Recommendation: 365 daily recommendation snapshots per asset group.
- D-06 Weather Data: 24 hourly readings per site × 12 sites = 288 records per engine run.
- D-07 Engine Config Constants: Single static config object.

## Acceptance Criteria

- **AC-1**: Synthetic data for all datasets covers a continuous rolling window of the past 12 months ending at the current date.
- **AC-2**: D-03a contains 96 intervals per asset at 15-minute resolution for the last 24 hours.
- **AC-3**: D-03b contains 672 intervals per asset (7 days × 96 per day) at 15-minute resolution.
- **AC-4**: D-04 contains 8,760 hourly price records for the past year.
- **AC-5**: D-05 contains 365 daily recommendation snapshots.
- **AC-6**: D-06 contains 288 records per engine run (24 hours × 12 sites).
- **AC-7**: All time series data uses UTC-offset timestamps (Europe/Amsterdam, CET/CEST).
- **AC-8**: No time series contains gaps, duplicate timestamps, or out-of-order entries.
