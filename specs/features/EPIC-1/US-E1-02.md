# US-E1-02 · Battery Master Data (D-01)

## User Story

**As a** developer building the PoC data layer, **I want to** provide a static registry of all 12 BESS assets with their physical and administrative properties, **so that** every downstream dataset and dashboard component can reference authoritative asset metadata.

## Description

Static registry of all BESS assets. One record per physical unit. Changes only when a new asset is commissioned or decommissioned. Does not include live telemetry. Stored in `battery_master.json`.

Each record contains: id, site_name, location, country, latitude, longitude, chemistry, power_rating_kw, capacity_kwh, duration_h, commissioned_date, manufacturer, model, withthegrid_node_id.

The fleet consists of 12 assets across Dutch cities. Eight assets are rated at 500 kW / 1000 kWh and four at 400 kW / 800 kWh, totalling 5,600 kW and 11,200 kWh (11.2 MWh).

## Acceptance Criteria

- **AC-1**: The dataset contains exactly 12 asset records.
- **AC-2**: Each record contains all required properties: id, site_name, location, country, latitude, longitude, chemistry, power_rating_kw, capacity_kwh, duration_h, commissioned_date, manufacturer, model, withthegrid_node_id.
- **AC-3**: Asset IDs follow the pattern "BESS-01" through "BESS-12".
- **AC-4**: Eight assets have 500 kW / 1000 kWh; four have 400 kW / 800 kWh.
- **AC-5**: Total fleet capacity sums to 11,200 kWh (11.2 MWh).
- **AC-6**: duration_h = capacity_kwh ÷ power_rating_kw for every asset.
- **AC-7**: All country values are "NL".
- **AC-8**: GPS coordinates are valid and within the Netherlands.
- **AC-9**: Each site_name is unique ("Site A" through "Site L").
- **AC-10**: Locations match the 12 specified Dutch cities.
