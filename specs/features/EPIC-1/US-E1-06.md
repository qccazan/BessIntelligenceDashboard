# US-E1-06 · AI Recommendation (D-05)

## User Story

**As a** developer building the PoC data layer, **I want to** provide a pre-computed AI recommendation object with portfolio-level and per-battery dispatch actions, **so that** the Market Forecast card (F-07) can display the recommendation block, explainability sentence, and per-battery action strip.

## Description

Pre-computed recommendation produced by F-06 engine. Stored in `ai_recommendation.json`. Contains: generated_at, portfolio_action, charge/discharge window start/end, charge/discharge price, price_spread_multiplier, avg_30d_spread_multiplier (hardcoded 6.1), confidence_pct, explanation, estimated_capture_eur, and per_battery_actions array with battery_id, action, window_start, window_end, and reason.

## Acceptance Criteria

- **AC-1**: All required top-level fields are present and non-null.
- **AC-2**: per_battery_actions contains exactly 12 entries matching D-01 assets.
- **AC-3**: Each per-battery action has battery_id, action, window_start, window_end, reason.
- **AC-4**: action is one of: "Charge", "Discharge", "Hold".
- **AC-5**: Fault-mode assets have action = "Hold" with reason "Fault — held offline".
- **AC-6**: charge_window corresponds to lowest-price hours in D-04; discharge_window to highest.
- **AC-7**: price_spread_multiplier = discharge_price / charge_price (within rounding).
- **AC-8**: avg_30d_spread_multiplier = 6.1 (from D-07).
- **AC-9**: confidence_pct is between 0 and 100.
- **AC-10**: Hold assets have window_start and window_end = "—".
- **AC-11**: explanation is non-empty natural-language text.
