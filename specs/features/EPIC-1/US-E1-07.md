# US-E1-07 · Weather Data (D-06)

## User Story

**As a** developer building the PoC data layer, **I want to** provide 24-hour hourly weather forecasts for each of the 12 site locations aligned with the D-04 price horizon, **so that** the F-06 engine can compute solar price adjustments, detect solar duck windows, and classify dispatch scenarios.

## Description

24-hour hourly weather forecast for each site, aligned with the D-04 price horizon. Stored in `weather_forecast.json` as an array of 288 records (24 hours × 12 sites). Critical: must be a time series, not a single snapshot. The engine computes avg_portfolio_ghi at each hour by averaging solar_irradiance_wm2 across all 12 sites.

Each record: site_id (matches D-01 site_name), location, hour_start (aligned with D-04 prices[].hour_start), ambient_temp_c, humidity_pct, wind_speed_ms, solar_irradiance_wm2 (GHI), cloud_cover_pct, condition.

NL design notes: April temps 8–15°C. Coastal sites 6–9 m/s wind; inland 3–5 m/s. At least one site > 8 m/s overnight. Solar irradiance = 0 between 20:00–06:00; peak GHI 09:00–15:00 of 400–480 W/m².

## Acceptance Criteria

- **AC-1**: The dataset contains 288 records (24 hours × 12 sites).
- **AC-2**: Each record has: site_id, location, hour_start, ambient_temp_c, humidity_pct, wind_speed_ms, solar_irradiance_wm2, cloud_cover_pct, condition.
- **AC-3**: site_id matches a valid site_name from D-01.
- **AC-4**: hour_start values exactly match D-04 prices[].hour_start for the same 24-hour horizon.
- **AC-5**: condition is one of: "clear", "partly_cloudy", "overcast", "rain", "storm".
- **AC-6**: solar_irradiance_wm2 = 0 for hours between 20:00 and 06:00 local time.
- **AC-7**: humidity_pct and cloud_cover_pct are between 0 and 100.
- **AC-8**: wind_speed_ms and solar_irradiance_wm2 are ≥ 0.
- **AC-9**: At least one coastal site has overnight wind > 8 m/s.
