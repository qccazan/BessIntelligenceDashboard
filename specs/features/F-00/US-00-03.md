## User Story

**As a** developer building the PoC data layer, **I want to** provide a live operational snapshot of each BESS asset with current telemetry readings, **so that** the Fleet Overview table, Current State card, and Portfolio Header KPIs can display real-time operational data.

## Description

Battery Real-Time Telemetry contains the live operational snapshot of each asset, polled or streamed at short intervals (e.g. every 5 seconds). Contains everything that can change between refreshes. Separate from master data so the two concerns do not mix. Stored in `battery_telemetry.json` as an array of one record per asset.

Each record contains: asset identifier (`id`, foreign key to D-01), timestamp of last reading, state of charge (`soc_pct`), state of health (`soh_pct`), instantaneous power (`power_kw`, positive = charging, negative = discharging), operational mode (`mode`: charging, discharging, idle, fault), internal temperature (`temperature_c`), DC bus voltage (`voltage_v`), DC current (`current_a`, positive = charging), next scheduled action (`next_action`: Charge, Discharge, Hold), next action time window (`next_action_window`), and active fault code (`fault_code`, null if no fault).

## Acceptance Criteria

- **AC-1**: The dataset contains exactly 12 telemetry records, one per asset defined in D-01.
- **AC-2**: Each record contains all required properties: `id`, `timestamp`, `soc_pct`, `soh_pct`, `power_kw`, `mode`, `temperature_c`, `voltage_v`, `current_a`, `next_action`, `next_action_window`, `fault_code`.
- **AC-3**: Each `id` matches a valid asset ID from D-01.
- **AC-4**: The `mode` field contains one of exactly four values: "charging", "discharging", "idle", or "fault".
- **AC-5**: When mode = "charging", `power_kw` is positive; when mode = "discharging", `power_kw` is negative; when mode = "idle" or "fault", `power_kw` is approximately zero.
- **AC-6**: The `soc_pct` value is between 0 and 100 (inclusive) for every asset.
- **AC-7**: The `soh_pct` value is between 0 and 100 (inclusive) for every asset.
- **AC-8**: The `fault_code` is non-null only when mode = "fault"; it is null for all other modes.
- **AC-9**: The `next_action` field contains one of: "Charge", "Discharge", or "Hold".
- **AC-10**: Fault-mode assets have `next_action` = "Hold" and `next_action_window` = "â€”".
- **AC-11**: The `timestamp` uses UTC-offset format (Europe/Amsterdam, CET/CEST).
