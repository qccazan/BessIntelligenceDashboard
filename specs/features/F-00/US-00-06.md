## User Story

**As a** developer building the PoC data layer, **I want to** provide a pre-computed AI recommendation object with portfolio-level and per-battery dispatch actions, **so that** the Market Forecast card can display the recommendation block, explainability sentence, and per-battery action strip.

## Description

The AI Recommendation is a pre-computed recommendation object produced by the intelligence layer. Consumed directly by the Market Forecast card. For the PoC this is static JSON â€” in production it would be regenerated each time a new price forecast is published. Stored in `ai_recommendation.json`.

The dataset contains: generation timestamp (`generated_at`), portfolio action label (`portfolio_action`), charge window start/end times, expected charge price (`charge_price_eur_mwh`), discharge window start/end times, expected discharge price (`discharge_price_eur_mwh`), price spread multiplier, model confidence (`confidence_pct`), natural-language explanation, estimated capture revenue (`estimated_capture_eur`), and an array of per-battery actions. Each per-battery action contains: battery ID, action (Charge/Discharge/Hold), window start time, and reason for deviation from portfolio default.

## Acceptance Criteria

- **AC-1**: The dataset contains all required top-level fields: `generated_at`, `portfolio_action`, `charge_window_start`, `charge_window_end`, `charge_price_eur_mwh`, `discharge_window_start`, `discharge_window_end`, `discharge_price_eur_mwh`, `price_spread_multiplier`, `confidence_pct`, `explanation`, `estimated_capture_eur`.
- **AC-2**: The `per_battery_actions` array contains exactly 12 entries, one per asset matching D-01.
- **AC-3**: Each per-battery action contains: `battery_id`, `action`, `window_start`, and `reason`.
- **AC-4**: The `action` field for each battery is one of: "Charge", "Discharge", or "Hold".
- **AC-5**: Any asset with D-02 mode = "fault" has action = "Hold" in per_battery_actions.
- **AC-6**: The `charge_window` corresponds to the lowest-price hours in D-04.
- **AC-7**: The `discharge_window` corresponds to the highest-price hours in D-04.
- **AC-8**: The `price_spread_multiplier` equals `discharge_price_eur_mwh Ă· charge_price_eur_mwh` (within rounding).
- **AC-9**: The `confidence_pct` is between 0 and 100 (inclusive).
- **AC-10**: The `explanation` field contains a non-empty natural-language string referencing concrete numbers.
- **AC-11**: The `estimated_capture_eur` is a positive number.
- **AC-12**: Per-battery `window_start` is "â€”" when action = "Hold".
