# US-00-02 · Battery Master Data (D-01)

## User Story

**As a** developer building the PoC data layer, **I want to** provide a static registry of all 12 BESS assets with their physical and administrative properties, **so that** every downstream dataset and dashboard component can reference authoritative asset metadata.

## Description

Battery Master Data is the static registry of all BESS assets. One record per physical unit. Changes only when a new asset is commissioned or decommissioned. Does not include live telemetry. Stored in `battery_master.json`.

Each record contains: unique asset identifier (`id`), human-readable site label (`site_name`), city (`location`), country code (`country`), GPS coordinates (`latitude`, `longitude`), battery cell chemistry (`chemistry`), maximum power (`power_rating_kw`), nameplate capacity (`capacity_kwh`), discharge duration (`duration_h`), commissioning date (`commissioned_date`), manufacturer, model, and Withthegrid node ID (`withthegrid_node_id`).

The fleet consists of 12 assets across Dutch cities: Amsterdam, Rotterdam, Utrecht, Eindhoven, Groningen, Tilburg, Almere, Breda, Nijmegen, Enschede, Haarlem, and Arnhem. Eight assets are rated at 500 kW / 1000 kWh and four at 400 kW / 800 kWh, totalling 5,600 kW and 11,200 kWh (11.2 MWh).

## Acceptance Criteria

- **AC-1**: The dataset contains exactly 12 asset records, one per physical BESS unit.
- **AC-2**: Each record contains all required properties: `id`, `site_name`, `location`, `country`, `latitude`, `longitude`, `chemistry`, `power_rating_kw`, `capacity_kwh`, `duration_h`, `commissioned_date`, `manufacturer`, `model`, `withthegrid_node_id`.
- **AC-3**: Asset IDs follow the pattern "BESS-01" through "BESS-12".
- **AC-4**: Eight assets have power_rating_kw = 500 and capacity_kwh = 1000; four assets have power_rating_kw = 400 and capacity_kwh = 800.
- **AC-5**: The total fleet capacity sums to 11,200 kWh (11.2 MWh).
- **AC-6**: The `duration_h` field equals `capacity_kwh ÷ power_rating_kw` for every asset.
- **AC-7**: All `country` values are "NL".
- **AC-8**: All `latitude` and `longitude` values are valid GPS coordinates within the Netherlands.
- **AC-9**: Each `site_name` is unique and follows the pattern "Site A" through "Site L".
- **AC-10**: Each `location` corresponds to a real Dutch city matching the asset table (Amsterdam, Rotterdam, Utrecht, Eindhoven, Groningen, Tilburg, Almere, Breda, Nijmegen, Enschede, Haarlem, Arnhem).
