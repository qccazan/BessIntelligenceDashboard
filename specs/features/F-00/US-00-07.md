# US-00-07 · Weather Data (D-06)

## User Story

**As a** developer building the PoC data layer, **I want to** provide hourly ambient environmental conditions for each of the 12 site locations, **so that** the system has thermal context for battery management and wind-driven price signal data for future features.

## Description

Weather Data contains ambient environmental conditions for each site location. Currently not rendered in the dashboard UI but influences two important backend concerns: (1) battery thermal management — ambient temperature affects safe operating limits; (2) dispatch context — high North Sea wind output directly suppresses overnight EPEX SPOT NL prices. Stored in `weather_forecast.json` with one entry per site, covering the same 24-hour horizon as D-04.

Each record contains: site identifier (`site_id`, matches `site_name` in D-01), city name (`location`), observation/forecast timestamp, ambient air temperature (`ambient_temp_c`), relative humidity (`humidity_pct`), wind speed at 10 m height (`wind_speed_ms`), global horizontal irradiance (`solar_irradiance_wm2`), cloud cover fraction (`cloud_cover_pct`), and summary condition label (`condition`: clear, partly_cloudy, overcast, rain, storm).

## Acceptance Criteria

- **AC-1**: The dataset contains weather records for all 12 sites defined in D-01.
- **AC-2**: Each record contains all required properties: `site_id`, `location`, `timestamp`, `ambient_temp_c`, `humidity_pct`, `wind_speed_ms`, `solar_irradiance_wm2`, `cloud_cover_pct`, `condition`.
- **AC-3**: Each `site_id` matches a valid `site_name` from D-01.
- **AC-4**: Each `location` matches the corresponding city from D-01 for the same site.
- **AC-5**: The `condition` field contains one of: "clear", "partly_cloudy", "overcast", "rain", or "storm".
- **AC-6**: The `humidity_pct` value is between 0 and 100 (inclusive).
- **AC-7**: The `cloud_cover_pct` value is between 0 and 100 (inclusive).
- **AC-8**: The `solar_irradiance_wm2` value is ≥ 0 (zero at night, positive during daylight).
- **AC-9**: The `wind_speed_ms` value is ≥ 0.
- **AC-10**: Weather data covers the same 24-hour horizon as D-04 (Day-Ahead Market Prices).
- **AC-11**: All timestamps use UTC-offset format (Europe/Amsterdam, CET/CEST).
