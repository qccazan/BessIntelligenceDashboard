# US-06-03 · Degradation Cost Modelling

## User Story

**As a** dispatch engine protecting battery longevity, **I want to** calculate the EUR cost of one cycle for each asset accounting for depth of discharge, C-rate stress, temperature, calendar aging, and SoH, **so that** the optimiser can compare gross revenue against battery wear and never recommend a cycle that costs more than it earns.

## Description

Battery replacement cost: 120 EUR/kWh (LFP pack, 2026 NL market). Expected cycle life at 80% DoD: 4,000 full-equivalent cycles. Base cost: 0.030 EUR/kWh per full equivalent cycle.

The degradation cost formula is: `cycle_deg_cost = 0.030 × capacity_kwh × DoD_factor × C_rate_factor × temp_penalty_factor × calendar_surcharge_factor × soh_cost_multiplier`

Where:
- DoD_factor = (DoD_pct / 100) ^ 1.3
- C_rate_factor = 1.0 + max(0, (C_rate − 0.5)) × 0.30
- temp_penalty_factor: 1.25 if < 5°C, 1.00 if 5–35°C, 1.20 if > 35°C
- calendar_surcharge_factor: 1.00 if < 10% time above 80% SoC, 1.08 if 10–25%, 1.15 if > 25%
- soh_cost_multiplier = 100 / soh_pct

## Acceptance Criteria

- **AC-1**: Base cycle cost is calculated as 120 / 4000 = 0.030 EUR/kWh.
- **AC-2**: DoD factor is calculated as (DoD_pct / 100) ^ 1.3 — e.g. DoD 40% → 0.333, DoD 80% → 0.742.
- **AC-3**: C-rate factor is 1.0 + max(0, (C_rate − 0.5)) × 0.30 — e.g. 0.5C → 1.00, 1.0C → 1.15.
- **AC-4**: Temperature penalty is 1.25 below 5°C, 1.00 between 5–35°C, 1.20 above 35°C.
- **AC-5**: Calendar aging surcharge is based on time-above-80% SoC: < 10% → 1.00, 10–25% → 1.08, > 25% → 1.15.
- **AC-6**: SoH cost multiplier is 100 / soh_pct — e.g. SoH 97% → 1.031, SoH 85% → 1.176.
- **AC-7**: The final cycle degradation cost is the product of all six factors × capacity_kwh.
- **AC-8**: NL April context: ambient temps 8–15°C, so temp_penalty_factor is always 1.00.
