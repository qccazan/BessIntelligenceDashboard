# US-06-04 · Window Optimisation

## User Story

**As a** dispatch engine selecting the optimal dispatch strategy, **I want to** score all valid charge/discharge window pairs by net benefit minus degradation cost and select the pair that maximises the portfolio benefit-to-wear ratio, applying a minimum profitability gate, **so that** the fleet only executes cycles when doing so is clearly profitable.

## Description

For each candidate window pair and each eligible asset, the engine computes: gross_revenue = (avg_discharge_price − avg_charge_price) × discharge_energy_kwh / 1000, then net_benefit = gross_revenue − cycle_deg_cost. The portfolio score = Σ net_benefit / Σ cycle_deg_cost (benefit-to-wear ratio). The optimal window pair is the argmax of this score. Tie-breaking: prefer the shorter charge window (shallower DoD = less wear) when scores are within 5%.

A minimum profitability gate requires portfolio_net_benefit ≥ 200 EUR. Below this threshold, the engine issues a Hold recommendation with an explanation citing low spread.

## Acceptance Criteria

- **AC-1**: Gross revenue is calculated as (avg_discharge_price − avg_charge_price) × discharge_energy_kwh / 1000 per asset per window pair.
- **AC-2**: Net benefit is gross revenue minus cycle degradation cost per asset.
- **AC-3**: Portfolio score is computed as sum(net_benefit) / sum(cycle_deg_cost) across eligible assets.
- **AC-4**: The optimal window pair is selected as the argmax of the portfolio score.
- **AC-5**: When two window pairs score within 5% of each other, the shorter charge window is preferred.
- **AC-6**: If portfolio_net_benefit < 200 EUR, the engine outputs portfolio_action = "Hold" with all per_battery_actions set to Hold.
- **AC-7**: When Hold is triggered, the explanation references the low spread as the reason.
- **AC-8**: Confidence_pct is still computed and reported even for Hold recommendations.
