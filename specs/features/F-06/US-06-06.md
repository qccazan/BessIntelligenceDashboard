# US-06-06 · Confidence Scoring

## User Story

**As a** dispatch engine quantifying recommendation reliability, **I want to** combine price forecast uncertainty, wind forecast error, solar forecast confidence, and fleet SoH variance into a single confidence percentage, **so that** the operator knows how much trust to place in the recommendation.

## Description

Confidence is computed from four weighted component scores using constants from D-07 (engine_config.json):

| Component | Weight | Calculation |
|-----------|--------|-------------|
| Price forecast accuracy | 40% | score = max(0, 100 − price_MAE_pct × 5) |
| Wind forecast confidence | 25% | score = max(0, 100 − wind_RMSE_ms × 8) |
| Solar forecast confidence | 20% | score = max(0, 100 − cloud_MAE_pct × 2). Defaults to 50 if no solar duck window was activated. |
| Fleet SoH uniformity | 15% | score = max(0, 100 − σ_SoH × 10) |

Final formula: `confidence_pct = round(0.40 × price_score + 0.25 × wind_score + 0.20 × solar_forecast_score + 0.15 × fleet_soh_score)`

Interpretation: 85%+ → high confidence (Wind+Solar, stable forecast). 65–84% → moderate. Below 65% → lower confidence, operator should review.

## Acceptance Criteria

- **AC-1**: Price forecast score is computed as max(0, 100 − price_MAE_pct × 5) using the D-07 constant.
- **AC-2**: Wind forecast score is computed as max(0, 100 − wind_RMSE_ms × 8) using the D-07 constant.
- **AC-3**: Solar forecast score is max(0, 100 − cloud_MAE_pct × 2) when a solar duck window is active; defaults to 50 otherwise.
- **AC-4**: Fleet SoH score is max(0, 100 − σ_SoH × 10) using the D-07 constant.
- **AC-5**: Confidence is the weighted sum: round(0.40 × price + 0.25 × wind + 0.20 × solar + 0.15 × soh).
- **AC-6**: The PoC mock dataset produces a confidence of 85% using adjusted D-07 constants (price_MAE_pct=4.0, wind_RMSE_ms=1.5).
- **AC-7**: Confidence is computed and reported even for Hold recommendations.
