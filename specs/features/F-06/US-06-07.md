# US-06-07 · Output Assembly

## User Story

**As a** dispatch engine producing consumable output, **I want to** assemble the complete D-05 AI Recommendation object with all fields populated from the preceding engine steps, including a natural-language explanation from a template, **so that** the Market Forecast card (F-07) can render the recommendation without any additional computation.

## Description

The engine assembles D-05 with: generated_at (current timestamp), portfolio_action (derived from majority action across per-battery assignments), charge/discharge windows and prices from Step 4, price_spread_multiplier (discharge_price / charge_price, rounded to 1 d.p.), avg_30d_spread_multiplier (from D-07, hardcoded 6.1), confidence_pct (from Step 6), estimated_capture_eur (sum of net_benefit across Charge/Discharge-assigned assets), and per_battery_actions array (from Step 5).

The explanation is generated from a template referencing: charge price, discharge price, spread multiplier, avg 30-day spread, wind context, optional solar context, spread comparison ("well above" / "above" / "in line with"), and estimated capture.

## Acceptance Criteria

- **AC-1**: The output contains all required D-05 fields: generated_at, portfolio_action, charge_window_start/end, charge_price_eur_mwh, discharge_window_start/end, discharge_price_eur_mwh, price_spread_multiplier, avg_30d_spread_multiplier, confidence_pct, explanation, estimated_capture_eur, per_battery_actions.
- **AC-2**: portfolio_action is derived from the majority action across per-battery assignments (e.g. "Coordinated charge").
- **AC-3**: price_spread_multiplier = discharge_price / charge_price, rounded to 1 decimal place.
- **AC-4**: avg_30d_spread_multiplier is copied from D-07 engine_config.json (hardcoded 6.1 for PoC).
- **AC-5**: estimated_capture_eur = sum of net_benefit across all Charge/Discharge-assigned assets.
- **AC-6**: per_battery_actions contains exactly 12 entries with battery_id, action, window_start, window_end, and reason.
- **AC-7**: The explanation is natural-language text referencing charge price, discharge price, spread multiplier, avg 30-day spread, wind context, and estimated capture.
- **AC-8**: Wind context in the explanation varies based on wind_speed_ms: > 7 → "high wind output across the North Sea", 5–7 → "sustained overnight wind generation", < 5 → "lower overnight demand".
- **AC-9**: Solar context is appended only for Wind+Solar scenarios.
- **AC-10**: Spread comparison is "well above", "above", or "in line with" based on current vs. 30-day average.
